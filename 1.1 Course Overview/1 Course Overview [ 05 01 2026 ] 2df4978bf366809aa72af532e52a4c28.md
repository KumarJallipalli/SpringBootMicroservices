## History of Programming Languages

- We have Computers in many forms ‚Üí Desktop Computers, Laptops, Smart Phones, Electronic Devices etc..
- Brain of Computer is Processor [ Processor, CPU, Micro Processor ‚Üí All refer to same thing ]
    - Image a Human in coma with all other body parts working is of NO Use
    - Similarly, Computer without CPU is of NO Use
- Processor/CPU is made up of Biliions of Transistors
- This Processor/CPU can only Understand 2 states
    - 0 ‚Üí NO Current flows
    - 1 ‚Üí Current flows
    - This is due to Transistors & this represents Binary [ 0 & 1 ]
    - So all the Operations like Addition, subtraction, etc.. are given in Binary Language called Machine Level Language [ MLL ]
- This MLL is very difficult to write, Understand & debug programs
    - A Program is a Set of Instructions given to a Computer to perform a Specific Task
- Hence, a New way of Programming is developed where instead of Binary codes, Nuemonics [ like SUB for Subtraction, ADD for Addition etc.. ] are used
    - As CPU can only understand Binaries,
        - An Assembler is developed to convert these neumonics to Binaries
    - This Language where Nuemonics are used to give instructions is called Assembly Level Language [ ALL ]
- This ALL also has many drawbacks just like MLL
- Hence, a New way of Programming is developed where instead of Nuemonics, actual English words & Symbols [ +, -, * etc.. ] are used
    - As CPU can only understand Binaries,
        - A Compiler  is developed to convert these English Words & Symbols to Binaries
    - This Language where English words & Symbols are used to give instructions is called High Level Language [ HLL ]
- In this way Modern Programming Languages called HLL are Developed

## Transistor

- A Transistor is¬†**a tiny semiconductor device**
    - They¬†**control the flow of electrical signals**, enabling the chip to process and store information.
    - How..?
        - A transistor consists of three terminals: the¬†**base**,¬†**collector**, and¬†**emitter**.
        - If there is¬†**no current**¬†at the base, the transistor acts like a closed switch ‚Üí No current flows between the collector and emitter.
        - If there is a¬†**current**¬†at the base, the transistor opens up ‚Üí Current flows through.
    - This ability to control electrical current allows transistors to work as a switch (switching things on and off) or as an amplifier (making signals stronger).
        - As a¬†**switch**: Transistors can rapidly turn on and off, representing binary states (0 and 1) that form the foundation of digital computing.
        - As an¬†**amplifier**: Transistors can also be used to boost weak electrical signals.
        - Hence, Transistors **acts as an electronic¬†switch¬†or¬†amplifier**
    - In digital electronics, transistors are used in large numbers to build¬†**logic gates**.
        - These form the foundation of computer processors.
        - By switching on and off very quickly, transistors help process the binary code that computers use to operate.

## BUS

- A computer bus is a **high-speed electronic pathway (a set of wires or circuits)** used to transfer data between components within a computer or between different computers
- Hence it is aka communication system
- Working:
    - When the CPU needs data from the HDD,
    - it sends a request through the System Bus to the chipset, which directs it to the HDD's interface bus (e.g., SATA).
    - The HDD then sends the requested data back along these buses to the CPU, often via RAM.
- **Key Functions:**
    - **Data Sharing:**¬†Moves actual information between components. ‚Üí Data BUS
    - **Addressing:**¬†Specifies where data should go or come from. ‚Üí Address BUS
    - **Control:**¬†Manages the timing and synchronization of operations. ‚Üí Control BUS
