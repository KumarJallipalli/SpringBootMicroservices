WKT,

- Static Variables will be Allocated with Memory & default values during Class Loading
- Static block will be Executed during Class Loading
- BUT, Static Methods are Executed only when they are Invoked Explicitly
- Instance block [ Non Static Blokc ] will be Executed during Objetc Creation && Before Constructor Execution
- Non-static field CANNOT be referenced from a static context
- BUT Static Variable/Method can be Referenced/Invoked from both [ Static & Non-Static ]
    - Static Variable/Method can be invoked using Class [ Name ] & Object variable/reference

```jsx
public class StaticKeyword {
    public static void main(String[] args) {
        Demo1 d1 = new Demo1();
        Demo1 d2 = new Demo1();
    }
}

class Demo1 {
    static {
        System.out.println("SB2");
    }
    static {
        System.out.println("SB1");
    }
    {
        System.out.println("NSB2");
    }
    {
        System.out.println("NSB1");
    }
}
```

Important distinction ⚠️

Here Error is caught by the **compiler**, NOT the JVM.

- Why Java forbids reading but allows writing 🤔
- This is a **language design decision**, not a JVM limitation.
- The JVM *would* technically know `a` exists
- But the **Java Language Specification** forbids forward reads

## The core rule (this is the money line) 💡

> **You may ASSIGN to a static variable before its declaration → Assignment ALLOWED** `a = 10;`
**but you may NOT READ it before its declaration. → Read NOT allowed** `System.out.println(a);`
> 

### If reading were allowed…

```java
static {
    System.out.println(a);
}
staticinta=10;
```

What should this print?

- `0`?
- `10`?
- Something else?

To avoid **confusing, order-dependent bugs**, Java says:

> “Nope. You must declare the variable *before* you read it.”
