## WKT

- If NO constructor is defined inside a Class
    - Java will invoke a default Constructor
    - This default Constructor is a Zero Parametarised Constructor
- Constructor Overloading is Possible
- Constructors can use access modifiers (e.g., `public`, `private`), but CANNOT be `abstract`, `static`, `final`, or `synchronized`.
- Constructor CANNOT be Inherited, But Parent Class Constructor can be Accessed using super keyword

## super in Constructors

```java
public class Constructor {
    public static void main(String[] args) {
        Demo d1 = new Demo(4);
        System.out.println(d1.num1);
        System.out.println(d1.num2);
    }
}

class Demo {
    int num1, num2;

    Demo () {
        System.out.println("Zero Parametarized Constructor");
    }
    Demo (int num1) {
        this();
        this.num1 = num1;
        System.out.println("One Parametarized Constructor");
    }
    Demo (int num1, int num2) {
        this.num1 = num1;
        this.num2 = num2;
        System.out.println("Two Parametarized Constructor");
    }

}
```

Here, 

- In the Zero Parametarized Constructor, `super()` super method call will be Invoked
- In the One Parametarized Constructor, `super()` super method call WON’T be Invoked [ due to `this()` this method call ]
- In the Two Parametarized Constructor, `super()` super method call will be Invoked

<aside>
💡

NOTE:

- No matter the type of the Constructor [ default, parametarized etc.. ]
    - The 1st statement of ANY CONSTRUCTOR will ALWYAS be `super()` [ super method call ]
    - Except when a Constructor is having a `this()` [ this method call ] as it’s 1st statement
</aside>

## Wrapper Classes

- Wrapper classes in Java are **object representations of primitive data types**
- **It encapsulates primitive data types within objects, so it can be used where objects are required.**
- Simpler → Wrapper classes convert primitive data types into objects so they can be used in object-based features of Java.
- Why Warpper Classes..?
    - **Since Java is object-oriented**, and some things only work with objects:
        - **Collections Framework**: Classes in the Java Collections Framework (e.g., `ArrayList`, `HashMap`) can only store objects, not primitives
        - **Null Values**: Unlike primitives, wrapper class objects can hold a `null` value, which is useful for representing the absence of data, such as in database operations.
        - **Utility Methods**: Each wrapper class provides various useful methods for data manipulation, type conversion, string parsing, and more (e.g., `Integer.parseInt()`, `Double.valueOf()`).
        - **Generics Support**: Generics in Java work exclusively with objects.
        - **Synchronization**: In multithreading, synchronization works with objects. Wrapper classes enable synchronization for primitive values.
    

```java
//        How to Define Wrapper Classes

        // Method - 1 : Creating a new Object using Integer Class --> Depreciated
        Integer i = new Integer(10);
        System.out.println(i);

        // Method - 2 : Using Static Method of Interger Class --> Recommended
        Integer j = Integer.valueOf(20);
        System.out.println(j);

        // Method - 3 : Autoboxing
        Integer k = 30;
        System.out.println(k);
        System.out.println(k instanceof Integer);
```

**Autoboxing and Unboxing**

Java simplifies working with wrapper classes through automatic conversions:

i.e., Java automatically converts between primitives and wrappers.

- **Autoboxing →** Automatic conversion of a primitive data type to its corresponding wrapper class object.
    - *Example*: `int x = 10; Integer myInt = x;`
- **Unboxing → A**utomatic conversion of a wrapper class object back to its corresponding primitive data type.
    - *Example*: `Integer myInt = 10; int x = myInt;`

These Autoboxing is used in Collections

```java
        ArrayList list = new ArrayList();
        list.add(10);       // 10 is primitive but autoboxed to Object
        list.add(20);
        list.add("John");   // String is a Class -> NO Autoboxing
        System.out.println(list);
```

| Primitive | Wrapper |
| --- | --- |
| Faster | Slightly slower |
| Less memory | More memory |
| Cannot be `null` | Can be `null` |
| No methods | Has methods |
